# Todo-API

Полноценное и готовое к работе REST API для управления списком дел. Проект написан на языке **Go** с использованием базы данных **SQLite3**.

---

## Архитектура и структура модулей

<pre>
├── main.go                  # Точка входа (Инициализация сервера и роутинг)
├── database/
│   └── database.go          # Подключение к БД и автоматические миграции
├── models/
│   └── todo.go              # модель (Структуры данных для Go/JSON/SQL)
├── repository/
│   └── todo_repository.go   # Слой данных (Изолированные SQL-запросы к БД)
└── controllers/
    └── todo_controller.go   # HTTP-слой (Парсинг запросов, валидация, ответы)
</pre>

## Эндпоинты

### 1.Получить все задачи
* **Метод:** `GET`
* **URL:** `/todos`
* **Query-параметры (Опционально):**
    * `search` (строка) — поиск по ключевому слову в названии.
    * `completed` (boolean) — фильтр по статусу (`true` или `false`).
* **Пример запроса:**

```bash
curl "http://localhost:8080/todos?search=кофе&completed=false"
```

### 2.Создать новую задачу
* **Метод:** `POST`
* **URL:** `/todos`
* **Тело запроса: {"title": "Написать README файл"}**
* **Пример запроса:**

```bash
curl "curl -X POST http://localhost:8080/todos -d '{"title":"Купить продукты"}'"
```

### 3.Обновить задачу
* **Метод:** `PATCH`
* **URL:** `/todos/{id}`
* **Тело запроса: Поддерживает изменение только title, только completed или всего вместе.**
* **Пример запроса:**

```bash
curl "curl -X PATCH http://localhost:8080/todos/1 -d '{"completed":true}'"
```

### 4.Удалить задачу
* **Метод:** `DELETE`
* **URL:** `/todos/{id}`
* **Пример запроса:**

```bash
curl "curl -X DELETE http://localhost:8080/todos/1"
```

* **Склонируй этот репозиторий себе на компьютер**

```bash
go run main.go
```
